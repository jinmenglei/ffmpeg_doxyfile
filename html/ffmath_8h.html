static av_always_inline double ff_exp10 ( double  x)
static

Compute 10^x for floating point values. Note: this function is by no means "correctly rounded", and is meant as a fast, reasonably accurate approximation. For instance, maximum relative error for the double precision variant is ~ 1e-13 for very small and very large values. This is ~2x faster than GNU libm's approach, which is still off by 2ulp on some inputs.

Compute x^y for floating point x, y. Note: this function is faster than the libm variant due to mainly 2 reasons:

  1. It does not handle any edge cases. In particular, this is only guaranteed to work correctly for x > 0.
  2. It is not as accurate as a standard nearly "correctly rounded" libm variant.
