\doxysubsection{Detailed Description}
Murmur\+Hash3 hash function implementation.

Murmur\+Hash3 is a non-\/cryptographic hash function, of which three incompatible versions were created by its inventor Austin Appleby\+:


\begin{DoxyItemize}
\item 32-\/bit output
\item 128-\/bit output for 32-\/bit platforms
\item 128-\/bit output for 64-\/bit platforms
\end{DoxyItemize}

F\+Fmpeg only implements the last variant\+: 128-\/bit output designed for 64-\/bit platforms. Even though the hash function was designed for 64-\/bit platforms, the function in reality works on 32-\/bit systems too, only with reduced performance.

\label{group__lavu__murmur3_lavu_murmur3_seedinfo}%
\Hypertarget{group__lavu__murmur3_lavu_murmur3_seedinfo}%
By design, Murmur\+Hash3 requires a seed to operate. In response to this, libavutil provides two functions for hash initiation, one that requires a seed (\mbox{\hyperlink{group__lavu__murmur3_ga8870f1e1d537790bc224c04e955d357b}{av\+\_\+murmur3\+\_\+init\+\_\+seeded()}}) and one that uses a fixed arbitrary integer as the seed, and therefore does not (\mbox{\hyperlink{group__lavu__murmur3_ga403fc40619831d043c444bf1df0b893d}{av\+\_\+murmur3\+\_\+init()}}).

To make hashes comparable, you should provide the same seed for all calls to this hash function -- if you are supplying one yourself, that is.
