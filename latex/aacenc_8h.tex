{\footnotesize\ttfamily const \mbox{\hyperlink{struct_a_a_c_p_c_e_info}{A\+A\+C\+P\+C\+E\+Info}} aac\+\_\+pce\+\_\+configs\mbox{[}$\,$\mbox{]}\hspace{0.3cm}{\ttfamily [static]}}

List of P\+CE (\mbox{\hyperlink{struct_program}{Program}} Configuration Element) for the channel layouts listed in channel\+\_\+layout.\+h

For those wishing in the future to add other layouts\+:


\begin{DoxyItemize}
\item num\+\_\+ele\+: number of elements in each group of front, side, back, lfe channels (an element is of type S\+CE (single channel), C\+PE (channel pair) for the first 3 groups; and is L\+FE for L\+FE group).
\item pairing\+: 0 for an S\+CE element or 1 for a C\+PE; does not apply to L\+FE group
\item index\+: there are three independent indices for S\+CE, C\+PE and L\+FE; they are incremented irrespective of the group to which the element belongs; they are not reset when going from one group to another

Example\+: for 7.\+0 channel layout, .pairing = \{ \{ 1, 0 \}, \{ 1 \}, \{ 1 \}, \}, (3 C\+PE and 1 S\+CE in front group) .index = \{ \{ 0, 0 \}, \{ 1 \}, \{ 2 \}, \}, (index is 0 for the single S\+CE but goes from 0 to 2 for the C\+P\+Es)

The index order impacts the channel ordering. But is otherwise arbitrary (the sequence could have been 2, 0, 1 instead of 0, 1, 2).

Spec allows for discontinuous indices, e.\+g. if one has a total of two S\+CE, S\+C\+E.\+0 S\+C\+E.\+15 is OK per spec; B\+UT it won\textquotesingle{}t be decoded by our A\+AC decoder which at this time requires that indices fully cover some range starting from 0 (S\+C\+E.\+1 S\+C\+E.\+0 is OK but not S\+C\+E.\+0 S\+C\+E.\+15).
\item config\+\_\+map\+: total number of elements and their types. Beware, the way the types are ordered impacts the final channel ordering.
\item reorder\+\_\+map\+: reorders the channels. 
\end{DoxyItemize}
