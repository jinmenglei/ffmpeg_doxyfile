\doxysubsection{Detailed Description}
F\+L\+AC parser

The F\+L\+AC parser buffers input until F\+L\+A\+C\+\_\+\+M\+I\+N\+\_\+\+H\+E\+A\+D\+E\+RS has been found. Each time it finds and verifies a C\+R\+C-\/8 header it sees which of the F\+L\+A\+C\+\_\+\+M\+A\+X\+\_\+\+S\+E\+Q\+U\+E\+N\+T\+I\+A\+L\+\_\+\+H\+E\+A\+D\+E\+RS that came before it have a valid C\+R\+C-\/16 footer that ends at the newly found header. Headers are scored by F\+L\+A\+C\+\_\+\+H\+E\+A\+D\+E\+R\+\_\+\+B\+A\+S\+E\+\_\+\+S\+C\+O\+RE plus the max of its crc-\/verified children, penalized by changes in sample rate, frame number, etc. The parser returns the frame with the highest score.
