Rescale a timestamp while preserving known durations.

This function is designed to be called per audio packet to scale the input timestamp to a different time base. Compared to a simple \mbox{\hyperlink{group__lavu__math_gaf02994a8bbeaa91d4757df179cbe567f}{av\+\_\+rescale\+\_\+q()}} call, this function is robust against possible inconsistent frame durations.

The {\ttfamily last} parameter is a state variable that must be preserved for all subsequent calls for the same stream. For the first call, {\ttfamily $\ast$last} should be initialized to \mbox{\hyperlink{group__lavu__time_ga2eaefe702f95f619ea6f2d08afa01be1}{A\+V\+\_\+\+N\+O\+P\+T\+S\+\_\+\+V\+A\+L\+UE}}.
