\doxysubsection{Detailed Description}
This structure describes decoded (raw) audio or video data.

\mbox{\hyperlink{struct_a_v_frame}{A\+V\+Frame}} must be allocated using \mbox{\hyperlink{group__lavu__frame_gac700017c5270c79c1e1befdeeb008b2f}{av\+\_\+frame\+\_\+alloc()}}. Note that this only allocates the \mbox{\hyperlink{struct_a_v_frame}{A\+V\+Frame}} itself, the buffers for the data must be managed through other means (see below). \mbox{\hyperlink{struct_a_v_frame}{A\+V\+Frame}} must be freed with \mbox{\hyperlink{group__lavu__frame_ga979d73f3228814aee56aeca0636e37cc}{av\+\_\+frame\+\_\+free()}}.

\mbox{\hyperlink{struct_a_v_frame}{A\+V\+Frame}} is typically allocated once and then reused multiple times to hold different data (e.\+g. a single \mbox{\hyperlink{struct_a_v_frame}{A\+V\+Frame}} to hold frames received from a decoder). In such a case, \mbox{\hyperlink{group__lavu__frame_ga0a2b687f9c1c5ed0089b01fd61227108}{av\+\_\+frame\+\_\+unref()}} will free any references held by the frame and reset it to its original clean state before it is reused again.

The data described by an \mbox{\hyperlink{struct_a_v_frame}{A\+V\+Frame}} is usually reference counted through the \mbox{\hyperlink{struct_a_v_buffer}{A\+V\+Buffer}} A\+PI. The underlying buffer references are stored in \mbox{\hyperlink{struct_a_v_frame_a4a64d05cc676bfa8e18bf22d16c8a51f}{A\+V\+Frame.\+buf}} / \mbox{\hyperlink{struct_a_v_frame_a254a144d113ceedc003ec1547cc5bd54}{A\+V\+Frame.\+extended\+\_\+buf}}. An \mbox{\hyperlink{struct_a_v_frame}{A\+V\+Frame}} is considered to be reference counted if at least one reference is set, i.\+e. if \mbox{\hyperlink{struct_a_v_frame_a4a64d05cc676bfa8e18bf22d16c8a51f}{A\+V\+Frame.\+buf}}\mbox{[}0\mbox{]} != N\+U\+LL. In such a case, every single data plane must be contained in one of the buffers in \mbox{\hyperlink{struct_a_v_frame_a4a64d05cc676bfa8e18bf22d16c8a51f}{A\+V\+Frame.\+buf}} or \mbox{\hyperlink{struct_a_v_frame_a254a144d113ceedc003ec1547cc5bd54}{A\+V\+Frame.\+extended\+\_\+buf}}. There may be a single buffer for all the data, or one separate buffer for each plane, or anything in between.

sizeof(\+A\+V\+Frame) is not a part of the public A\+BI, so new fields may be added to the end with a minor bump.

Fields can be accessed through A\+V\+Options, the name string used, matches the C structure field name for fields accessible through A\+V\+Options. The \mbox{\hyperlink{struct_a_v_class}{A\+V\+Class}} for \mbox{\hyperlink{struct_a_v_frame}{A\+V\+Frame}} can be obtained from \mbox{\hyperlink{group__lavc__core_gad55dfc8e9e168e498ab053b38ba6dec8}{avcodec\+\_\+get\+\_\+frame\+\_\+class()}}

For planar audio with more than A\+V\+\_\+\+N\+U\+M\+\_\+\+D\+A\+T\+A\+\_\+\+P\+O\+I\+N\+T\+E\+RS channels, there may be more buffers than can fit in this array.
