Retrieve one cached packet from the rm-\/context. The real container can store several packets (as interpreted by the codec) in a single container packet, which means the demuxer holds some back when the first container packet is parsed and returned. The result is that rm-\/$>$audio\+\_\+pkt\+\_\+cnt is a positive number, the amount of cached packets. Using this function, each of those packets can be retrieved sequentially.
