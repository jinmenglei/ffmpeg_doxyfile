Scale one horizontal line of input data using a bilinear filter to produce one line of output data. Compared to Sws\+Context-\/$>$h\+Scale(), please take note of the following caveats when using these\+:
\begin{DoxyItemize}
\item Scaling is done using only 7 bits instead of 14-\/bit coefficients.
\item You can use no more than 5 input pixels to produce 4 output pixels. Therefore, this filter should not be used for downscaling by more than $\sim$20\% in width (because that equals more than 5/4th downscaling and thus more than 5 pixels input per 4 pixels output).
\item In general, bilinear filters create artifacts during downscaling (even when $<$20\%), because one output pixel will span more than one input pixel, and thus some pixels will need edges of both neighbor pixels to interpolate the output pixel. Since you can use at most two input pixels per output pixel in bilinear scaling, this is impossible and thus downscaling by any size will create artifacts. To enable this type of scaling, set S\+W\+S\+\_\+\+F\+L\+A\+G\+\_\+\+F\+A\+S\+T\+\_\+\+B\+I\+L\+I\+N\+E\+AR in Sws\+Context-\/$>$flags.
