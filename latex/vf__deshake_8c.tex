\doxysubsection{Detailed Description}
fast deshake / depan video filter

S\+AD block-\/matching motion compensation to fix small changes in horizontal and/or vertical shift. This filter helps remove camera shake from hand-\/holding a camera, bumping a tripod, moving on a vehicle, etc.

Algorithm\+:
\begin{DoxyItemize}
\item For each frame with one previous reference frame
\begin{DoxyItemize}
\item For each block in the frame
\begin{DoxyItemize}
\item If contrast $>$ threshold then find likely motion vector
\end{DoxyItemize}
\item For all found motion vectors
\begin{DoxyItemize}
\item Find most common, store as global motion vector
\end{DoxyItemize}
\item Find most likely rotation angle
\item \mbox{\hyperlink{struct_transform}{Transform}} image along global motion
\end{DoxyItemize}
\end{DoxyItemize}

T\+O\+DO\+:
\begin{DoxyItemize}
\item Fill frame edges based on previous/next reference frames
\item Fill frame edges by stretching image near the edges?
\begin{DoxyItemize}
\item Can this be done quickly and look decent?
\end{DoxyItemize}
\end{DoxyItemize}

Dark Shikari links to \href{http://wiki.videolan.org/SoC_x264_2010\#GPU_Motion_Estimation_2}{\texttt{ http\+://wiki.\+videolan.\+org/\+So\+C\+\_\+x264\+\_\+2010\#\+G\+P\+U\+\_\+\+Motion\+\_\+\+Estimation\+\_\+2}} for an algorithm similar to what could be used here to get the gmv It requires only a couple diamond searches + fast downscaling

Special thanks to Jason Kotenko for his help with the algorithm and my inability to see simple errors in C code.
