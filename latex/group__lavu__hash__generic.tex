\doxysubsection{Detailed Description}
An abstraction layer for all hash functions supported by libavutil.

If your application needs to support a wide range of different hash functions, then the Generic Hashing A\+PI is for you. It provides a generic, reusable A\+PI for \mbox{\hyperlink{group__lavu__hash}{all hash functions}} implemented in libavutil. If you just need to use one particular hash function, use the \mbox{\hyperlink{group__lavu__hash}{individual hash}} directly.\hypertarget{group__lavu__hash__generic_Sample}{}\doxysubsection{Code}\label{group__lavu__hash__generic_Sample}

Maximum value that \mbox{\hyperlink{group__lavu__hash__generic_ga8bf8f94e3ebf73f838d017f83352d880}{av\+\_\+hash\+\_\+get\+\_\+size()}} will currently return.

You can use this if you absolutely want or need to use static allocation for the output buffer and are fine with not supporting hashes newly added to libavutil without recompilation.

\begin{DoxyWarning}{Warning}
Adding new hashes with larger sizes, and increasing the macro while doing so, will not be considered an A\+BI change. To prevent your code from overflowing a buffer, either dynamically allocate the output buffer with \mbox{\hyperlink{group__lavu__hash__generic_ga8bf8f94e3ebf73f838d017f83352d880}{av\+\_\+hash\+\_\+get\+\_\+size()}}, or limit your use of the Hashing A\+PI to hashes that are already in F\+Fmpeg during the time of compilation. 
\end{DoxyWarning}

Finalize a hash context and store the hexadecimal representation of the actual hash value as a string.
