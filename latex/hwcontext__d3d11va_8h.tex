\doxysubsection{Detailed Description}
An A\+P\+I-\/specific header for A\+V\+\_\+\+H\+W\+D\+E\+V\+I\+C\+E\+\_\+\+T\+Y\+P\+E\+\_\+\+D3\+D11\+VA.

The default pool implementation will be fixed-\/size if initial\+\_\+pool\+\_\+size is set (and allocate elements from an array texture). Otherwise it will allocate individual textures. Be aware that decoding requires a single array texture.

Using sw\+\_\+format==A\+V\+\_\+\+P\+I\+X\+\_\+\+F\+M\+T\+\_\+\+Y\+U\+V420P has special semantics, and maps to D\+X\+G\+I\+\_\+\+F\+O\+R\+M\+A\+T\+\_\+420\+\_\+\+O\+P\+A\+Q\+UE. \mbox{\hyperlink{hwcontext_8c_abf1b1664b8239d953ae2cac8b643815a}{av\+\_\+hwframe\+\_\+transfer\+\_\+data()}} is not supported for this format. Refer to M\+S\+DN for details.

\mbox{\hyperlink{hwcontext_8c_a21fbd088225e4e25c4d9a01b3f5e8c51}{av\+\_\+hwdevice\+\_\+ctx\+\_\+create()}} for this device type supports a key named \char`\"{}debug\char`\"{} for the \mbox{\hyperlink{struct_a_v_dictionary}{A\+V\+Dictionary}} entry. If this is set to any value, the device creation code will try to load various supported D3D debugging layers.
