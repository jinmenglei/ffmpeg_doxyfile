Compute 10$^\wedge$x for floating point values. Note\+: this function is by no means \char`\"{}correctly rounded\char`\"{}, and is meant as a fast, reasonably accurate approximation. For instance, maximum relative error for the double precision variant is $\sim$ 1e-\/13 for very small and very large values. This is $\sim$2x faster than G\+NU libm\textquotesingle{}s approach, which is still off by 2ulp on some inputs.
