\doxysubsection{Detailed Description}
Autodesk Animator F\+L\+I/\+F\+LC Video \mbox{\hyperlink{struct_decoder}{Decoder}} by Mike Melanson (\href{mailto:melanson@pcisys.net}{\texttt{ melanson@pcisys.\+net}}) for more information on the .fli/.flc file format and all of its many variations, visit\+: \href{http://www.compuphase.com/flic.htm}{\texttt{ http\+://www.\+compuphase.\+com/flic.\+htm}}

This decoder outputs P\+A\+L8/\+R\+G\+B555/\+R\+G\+B565/\+B\+G\+R24. To use this decoder, be sure that your demuxer sends the F\+LI file header to the decoder via the extradata chunk in \mbox{\hyperlink{struct_a_v_codec_context}{A\+V\+Codec\+Context}}. The chunk should be 128 bytes large. The only exception is for F\+LI files from the game \char`\"{}\+Magic Carpet\char`\"{}, in which the header is only 12 bytes.
