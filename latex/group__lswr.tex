\doxysubsection{Detailed Description}
Audio resampling, sample format conversion and mixing library.

Interaction with lswr is done through \mbox{\hyperlink{struct_swr_context}{Swr\+Context}}, which is allocated with \mbox{\hyperlink{group__lswr_gaf58c4ff10f73d74bdab8e5aa7193147c}{swr\+\_\+alloc()}} or \mbox{\hyperlink{group__lswr_gad7f29022726c871aadf935281a9813bc}{swr\+\_\+alloc\+\_\+set\+\_\+opts()}}. It is opaque, so all parameters must be set with the \mbox{\hyperlink{group__avoptions}{A\+V\+Options}} A\+PI.

The first thing you will need to do in order to use lswr is to allocate \mbox{\hyperlink{struct_swr_context}{Swr\+Context}}. This can be done with \mbox{\hyperlink{group__lswr_gaf58c4ff10f73d74bdab8e5aa7193147c}{swr\+\_\+alloc()}} or \mbox{\hyperlink{group__lswr_gad7f29022726c871aadf935281a9813bc}{swr\+\_\+alloc\+\_\+set\+\_\+opts()}}. If you are using the former, you must set options through the \mbox{\hyperlink{group__avoptions}{A\+V\+Options}} A\+PI. The latter function provides the same feature, but it allows you to set some common options in the same statement.

For example the following code will setup conversion from planar float sample format to interleaved signed 16-\/bit integer, downsampling from 48k\+Hz to 44.\+1k\+Hz and downmixing from 5.\+1 channels to stereo (using the default mixing matrix). This is using the \mbox{\hyperlink{group__lswr_gaf58c4ff10f73d74bdab8e5aa7193147c}{swr\+\_\+alloc()}} function.

Once all values have been set, it must be initialized with \mbox{\hyperlink{group__lswr_gae173e8ed91717700471a1dcd06f00f67}{swr\+\_\+init()}}. If you need to change the conversion parameters, you can change the parameters using A\+V\+Options, as described above in the first example; or by using \mbox{\hyperlink{group__lswr_gad7f29022726c871aadf935281a9813bc}{swr\+\_\+alloc\+\_\+set\+\_\+opts()}}, but with the first argument the allocated context. You must then call \mbox{\hyperlink{group__lswr_gae173e8ed91717700471a1dcd06f00f67}{swr\+\_\+init()}} again.

The conversion itself is done by repeatedly calling \mbox{\hyperlink{group__lswr_gaa5bb6cab830146efa8c760fa66ee582a}{swr\+\_\+convert()}}. Note that the samples may get buffered in swr if you provide insufficient output space or if sample rate conversion is done, which requires \char`\"{}future\char`\"{} samples. Samples that do not require future input can be retrieved at any time by using \mbox{\hyperlink{group__lswr_gaa5bb6cab830146efa8c760fa66ee582a}{swr\+\_\+convert()}} (in\+\_\+count can be set to 0). At the end of conversion the resampling buffer can be flushed by calling \mbox{\hyperlink{group__lswr_gaa5bb6cab830146efa8c760fa66ee582a}{swr\+\_\+convert()}} with N\+U\+LL in and 0 in\+\_\+count.

The samples used in the conversion process can be managed with the libavutil \mbox{\hyperlink{group__lavu__sampmanip}{samples manipulation}} A\+PI, including \mbox{\hyperlink{group__lavu__sampmanip_ga4db4c77f928d32c7d8854732f50b8c04}{av\+\_\+samples\+\_\+alloc()}} function used in the following example.

The delay between input and output, can at any time be found by using \mbox{\hyperlink{group__lswr_ga5121a5a7890a2d23b72dc871dd0ebb06}{swr\+\_\+get\+\_\+delay()}}.

When the conversion is finished, the conversion context and everything associated with it must be freed with \mbox{\hyperlink{group__lswr_ga818f7d78b1ad7d8d5b70de374b668c34}{swr\+\_\+free()}}. A \mbox{\hyperlink{group__lswr_gaa4bf1048740dfc08d68aba9f1b4db22e}{swr\+\_\+close()}} function is also available, but it exists mainly for compatibility with libavresample, and is not required to be called.

There will be no memory leak if the data is not completely flushed before \mbox{\hyperlink{group__lswr_ga818f7d78b1ad7d8d5b70de374b668c34}{swr\+\_\+free()}}.

Gets the delay the next input sample will experience relative to the next output sample.

Swresample can buffer data if more input has been provided than available output space, also converting between sample rates needs a delay. This function returns the sum of all such delays. The exact delay is not necessarily an integer value in either input or output sample rate. Especially when downsampling by a large value, the output sample rate may be a poor choice to represent the delay, similarly for upsampling and the input sample rate.
